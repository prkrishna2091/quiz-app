# API and Functionality Documentation

## 1) Functional Modules

## A. Authentication Module
- **Goal**: Convert Google identity token into application JWT.
- **Flow**:
  1. UI receives Google ID token.
  2. UI calls backend `/api/auth/google` with token.
  3. Backend verifies Google token.
  4. Backend returns app JWT.
  5. UI sends JWT in `Authorization: Bearer <jwt>` for protected APIs.

## B. Quiz Generation Module
- **Goal**: Build quiz from YouTube video.
- **Flow**:
  1. Client sends YouTube URL to `/api/quiz/generate`.
  2. App downloads YouTube metadata + captions.
  3. App sends cleaned transcript to Gemini.
  4. App returns combined payload (caption + generated quiz + metadata).

## C. Health and Monitoring Module
- **Goal**: Operational visibility.
- Exposes application health and build metadata via `/health`.

## 2) Endpoint Reference

## `POST /api/auth/google`

### Description
Verifies Google ID token and issues application JWT.

### Request Body
```json
{
  "token": "<google-id-token>"
}
```

### Success Response (200)
```json
{
  "token": "<app-jwt-token>"
}
```

### Error Response (401)
Empty body (as implemented in controller).

---

## `POST /api/quiz/generate`

### Auth
Requires `Authorization: Bearer <app-jwt-token>`.

### Request Body
```json
{
  "youtubeUrl": "https://www.youtube.com/watch?v=VIDEO_ID"
}
```

### Success Response (200)
```json
{
  "caption": "cleaned transcript text...",
  "quiz": "{...json generated by model as string...}",
  "videoTitle": "Video title",
  "channelName": "Channel name",
  "videoLength": "12:34"
}
```

### Validation / Error Cases
- Missing URL -> 400
- Missing request body -> 400
- Caption extraction failure -> 400 (wrapped runtime)
- AI generation failure -> 400 (wrapped runtime)

## 3) Example API Calls

```powershell
# 1) Google login
curl -X POST "http://localhost:8451/api/auth/google" `
  -H "Content-Type: application/json" `
  -d '{"token":"<google-id-token>"}'

# 2) Quiz generation (protected)
curl -X POST "http://localhost:8451/api/quiz/generate" `
  -H "Content-Type: application/json" `
  -H "Authorization: Bearer <app-jwt-token>" `
  -d '{"youtubeUrl":"https://www.youtube.com/watch?v=UabBYexBD4k"}'

# 3) Health
curl "http://localhost:8451/health"
```

## 4) Security Behavior Summary

- JWT is extracted only from `Authorization` header using `Bearer` scheme.
- Public endpoints are intentionally whitelisted in security config.
- Any non-whitelisted endpoint requires a valid JWT.
- Invalid/missing JWT leads to authentication failure for protected routes.
